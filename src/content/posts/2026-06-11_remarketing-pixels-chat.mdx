## How do you fire remarketing pixels from ecommerce chat behavior?

A shopper who asks "waterproof hiking boots under $150" has declared a category, a feature, and a price point. That's a retargeting signal. If your pixel only fires on page loads, that intent disappears the moment they close the tab. You can only retarget on signals your pixel actually captured. Firing on chat intent requires attaching pixel calls to specific assistant events: query submitted, product viewed, conversation closed. Category interest has a no-code toggle in the dashboard; product view and conversation abandonment require Nobi's Hooks API.

## Why is in-assistant behavior a stronger retargeting signal than a page view?

A page view tells you a visitor landed somewhere. A chat query tells you what they wanted, whether they found it, and exactly where they dropped off - three qualification layers a page-view pixel collapses into one undifferentiated audience.

Page-level retargeting audiences mix high-intent visitors with three-second bounces, accidental clicks, and returning customers who already converted. A visitor who submits two category queries and closes the assistant without adding to cart is behaviorally distinct from a homepage bounce - your pixel should tag them into different ad sets.

DTC brands running paid retargeting consistently see their highest ROAS from mid-funnel audiences - shoppers who expressed intent but didn't convert - and a chat query is a precise mid-funnel signal by definition. Page-only pixels also miss zero-result moments: A shopper may ask for something you carry but phrase it in a way your site can't surface. That's a retargetable moment a page view would never capture.

## How do you fire pixels on category interest, purchase intent, and conversation abandonment?

With that developer capacity confirmed, map each meaningful assistant event to a pixel custom event name and attach the pixel call inside the Nobi Hooks callback for that action. The three events worth tagging first: a submitted category query (interest), a product card click inside the assistant (intent), and a conversation close without an add-to-cart in the same session (abandonment).

For category-interest signals, Nobi has a no-code path through the dashboard:

1. Navigate to **Assistant → Behavior** in your Nobi dashboard
2. Scroll down to the **Remarketing** section
3. Toggle on **Send search data to Facebook Pixel** and/or **Send search data to Google Tag Manager** as needed
4. Each toggle saves immediately - no separate save button

The Facebook toggle fires a standard Search event with the shopper's query in the `search_string` parameter. The GTM toggle pushes a `Nobi_search` event to the dataLayer with the query in `Nobi_search_query`.

Product view and abandonment signals require Hooks. Intercept `onProductView` - it fires when a shopper clicks a product card inside the assistant - and call a ViewContent event with the SKU, price, and category. That populates your mid-funnel purchase-intent audience. Intercept `onConversationClose` and check whether an add-to-cart fired in the same session; if not, fire a custom `ChatAbandoned` event. Shoppers in that pool engaged with the assistant and left without buying - typically the sharpest retargeting seed in the whole stack.

One signal worth using as a suppression trigger rather than a retargeting one: query override responses. When a shopper asks a return-policy or warranty question and gets a pinned verbatim answer, they're likely an existing customer. Tag those events and exclude that audience from acquisition campaigns - they don't need prospecting creative.

The cleanest implementation pattern: push one custom dataLayer event from each Hooks callback, then configure a single GTM trigger listening to that event to fire all your platform pixels. Hooks logic stays in one place; you're not duplicating the callback per platform.

## How do you connect Meta, Google, and TikTok pixels to Nobi's chat events?

All three platforms read from the same JavaScript event layer Nobi exposes through its Hooks API. One GTM tag listens for Nobi Hooks events and routes the payload to each pixel's tracking call - the platform-specific syntax differs, but the underlying data source is the same.

**Meta Pixel.** On `onQuery`, fire a custom `ChatCategoryInterest` event with the shopper's query string as `content_category`. On `onProductView`, fire a standard `ViewContent` event with the SKU and `content_type: 'product'`. On `onConversationClose` with no purchase event in the session, fire `ChatAbandoned`.

iOS browser pixel match rates have declined since App Tracking Transparency, so consider pairing the browser pixel with Meta's Conversions API (CAPI). Forward the same three events through your own backend to the CAPI endpoint, and set deduplication keys on both paths so the same shopper doesn't inflate event counts from two sources.

**Google Ads.** On `onQuery`, send a `chat_category_interest` event via `gtag` with your conversion ID. In Google Ads Audience Manager, define the audience rule to match `event equals chat_category_interest`. Repeat for `chat_product_view` and `chat_abandoned` - each maps to its own remarketing pool.

**TikTok Pixel.** On `onProductView`, fire `ViewContent` with the SKU. On `onConversationClose` without an add-to-cart, fire `ChatAbandoned`. One constraint worth planning for: TikTok requires at least 100 qualifying event fires before an audience is usable in Ads Manager. On lower-volume stores, expect a one- to two-week warm-up period before you can activate it.

For the GTM setup, create one custom HTML tag that registers the Nobi Hooks listener and pushes structured dataLayer events. Then build separate tag-and-trigger pairs for Meta, Google, and TikTok that fire off that dataLayer event. Any change to the event payload happens in one place and propagates to all three platforms automatically.

## How do you build retargeting audiences from chat assistant behavior?

With events firing across Meta, Google, and TikTok from the same GTM payload, the next step is building separate audiences matched to specific assistant-behavior events - not broad page-visit rules. The three highest-value segments are category-intent visitors (queried but didn't buy), product-view abandoners (clicked a product card but didn't add to cart), and conversation completers (asked a question and got a full answer but still didn't convert).

**Meta Ads Manager.** Open Custom Audiences, choose Website, and select "People who triggered a specific event." Pick ChatCategoryInterest as the event. Add a Purchase exclusion in the same window to filter out existing customers who already converted - this keeps the audience focused on non-converters you can still win.

**Google Ads.** In Audience Manager, create a Website visitors rule using "Visitors of a page who also triggered a specific tag event" and select chat_category_interest. Build a second audience using chat_abandoned - that's your highest-intent non-converter layer, the shoppers who engaged with the assistant and left without buying.

Set a seven- to fourteen-day lookback window for most categories. Intent signals decay quickly - a shopper who asked about running shoes last month is much less likely to still be in market. Reserve a thirty-day window for high-consideration product lines like furniture, footwear, and outerwear, where purchase cycles run longer.

Build a suppression audience from shoppers whose queries triggered a return-policy, WISMO (where-is-my-order), or order-status response. Those are existing customers, not acquisition targets. Running prospecting creative against them wastes budget and inflates your CAC.

One more audience worth building: lookalikes seeded from shoppers who fired ChatAbandoned and then converted in a separate purchase-attributed cohort. These outperform lookalikes seeded from all purchasers because the seed audience filters for demonstrated product intent before the purchase happened.

## How do you measure retargeting ROAS from chat-derived pixel signals?

Cleaner signal in the seed audience only proves its value against a baseline. Measurement requires a clean thirty-day pre-launch ROAS figure from your existing site-visitor retargeting on all three platforms - that's the comparison line you hold chat-intent audiences against once they're live. Without it, there's no way to tell whether ROAS improved because of the chat signal or because you were already reaching those shoppers through standard page pixels.

When you activate the chat audiences, put them in a separate ad set at the same budget split and the same creative. Let the platforms report them side by side for two weeks before drawing conclusions.

Use CPM as a quality check while you wait. If chat-intent audiences have a lower CPM than page-visitor audiences at comparable ROAS, the two pools overlap heavily - you're paying twice for the same people. If CPM is higher and ROAS is also higher, the intent filter is pulling its weight.

Cross-reference Nobi's CVR and revenue attribution dashboard with your ad platform's conversion windows. If Nobi attributes a sale to an assistant interaction and Meta attributes the same sale to a retargeting click, one of those windows is too wide. Narrow the shorter view-through window first.

In our experience, chat-intent audiences tend to convert at 1.5x to 2x the rate of broad page-visitor retargeting in the same category - though results vary by category and catalog size. At parity, the audience definition is too broad - walk it back to ChatAbandoned only before expanding to category-interest segments.

One honest limit: Nobi's reporting focuses on search and CVR metrics, not on why shoppers abandoned inside the assistant. For a full picture of drop-off behavior, run a session-recording tool alongside these pixel events.

## Frequently asked questions about remarketing pixels and ecommerce chat

**Does this work on Shopify?**
Yes. Nobi deploys via a small Shopify theme edit. The Hooks listener can go in the theme's JavaScript file or inside an existing GTM container you already have on the store.

**What if Meta or TikTok audiences are too small to serve ads?**
TikTok requires 100 qualifying event fires and Meta requires at least 100 matched visitors before an audience activates. On lower-volume stores, combine ChatCategoryInterest and ChatAbandoned into one audience until it clears the threshold, then split once volume allows.

**Does this work with a server-side pixel setup?**
Yes - forward the same conversation-event data to your CAPI or Google server-side tagging endpoint from your own backend. That also reduces how much your audiences depend on browser-side cookie consent, which helps addressable audience size in regions with strict cookie rules.

**What about WISMO, return, and exchange queries?**
Tag these as suppression events, not acquisition signals. A shopper asking where their order is has already converted. Excluding them from prospecting campaigns saves more budget than retargeting them would earn.

**Does Nobi have a native pixel connector?**
Partially. Category interest signals have a no-code path: toggle on the Facebook Pixel or GTM integration in Assistant → Behavior → Remarketing and those events fire automatically. Product view and abandonment signals require the Hooks API, which needs a developer or someone comfortable writing GTM custom HTML tags. That's a one-time setup - once the listener is in place it runs without ongoing maintenance.
